Matching atleast 2a's.

{n} = n times
{m,n} = at least m times but no more than n times
{m,} = at least m times

a+: from once to infinity
a?: matches the a if it exists but only the empty string if it does not.
a*: 0 to initinity empty string is also matched
a{0,}: matches 0 to infinity also the empty string

* = {0, }
+ = {1,}
?= {0,1} may be nothing is matched

/regexp?/g
matches both regex and regexp as p is optional

4) /<.+?>/g
<p>foo</p>
matches only the <p> but if "?" removed it will match the whole string as it tries to match as much as it can in a string.

5) /[abc]+/
matches any number of these characters

6) /[a-z]+/
matches all the letters. Does not match numbers or symbols.

7) /[a-z0-9_]+/
matches all the letters&numbers and underscore.

8) \w means : letters, numbers and underscore both upper and lower cases. Not unicode aware, i.e not greek letters (no whitespace).

\w = [a-zA-Z0-9_]
\w- = word char class + hyphen
\d = [0-9]  any interger no decimals
\s ~ [\t\r\n ]    any kind of whitespace and unicode aware. Any weird whitespace that unicode contains.
\w+ = any \w multiple chars.

\W = [^\w] matches negation of letters, numbers or underscore.
      Negation
\D = [^\d] matches negation of any integer
      Negation
\S = [^\s] matches everything but whitespace.

Ex: /^#([a-f\d]{3}){1,2}$/g
matches :
"#cdfFFa6" any 3 or 6 digit hex number.

9) Paranthesis is grouping queries and | is for or'ing.
Paranthesis is also capturing not only grouping

Ex: /c(ab|ba)/g -->> either ab or ba is matched.afads
Say we have a string: cab . The above rule will match and store both
cab an ab.

Ex: /(JAVA|ECMA)Script/g
"JAVASCRIPT"
