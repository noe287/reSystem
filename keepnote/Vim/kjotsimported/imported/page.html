<p><span style="font-family:Ubuntu;">16.2. Create new branch</span></p> 

<p><p>&nbsp;You can create a new branch via the  <span style="font-family:Courier New,courier;">git branch [newname]</span><span style="font-family:Ubuntu;"> command. This command allows optionally to specify the commit id, if not specified the currently checked out commit will be used to create the branch.</span></p>

<p><span style="font-family:Courier New,courier;"><span style="font-size:11pt;"><em># &lt;hash&gt; in the above is optional </em></span></span></p> 

<p>git branch testing</p> 

<p>&nbsp;<p><em># Switch to your new branch</em></p> 

<p>git checkout testing</p>

<p>git </p> 

<p>&nbsp;<p>checkout master</p> 

<p>&nbsp;<p><em># Check that the content of test01 is the old one</em> cattest01        </p>

<p>To create a branch and to switch to it at the same time you can use the  <span style="font-family:Courier New,courier;">git checkout</span><span style="font-family:Ubuntu;"> command with the  </span><span style="font-family:Courier New,courier;"><em>-b</em></span><span style="font-family:Ubuntu;"><em> </em>parameter.</span></p> 

<p><span style="font-family:Monaco,Courier,monospace;"><span style="font-size:-1pt;"><em># Create branch and switch to it</em></span></span></p> 

<p><span style="font-family:Courier New,courier;"><span style="font-size:11pt;">git checkout -b bugreport12</span></span></p>

<p><span style="font-family:Monaco,Courier,monospace;"><span style="font-size:-1pt;"><em># rename branch</em></span></span></p>

<p><span style="font-size:14pt;"><strong>17. Differences between branches</strong></span></p> 

<p><span style="font-size:11pt;">To see the difference between two branches you can use the following command.</span></p> 

<p> <span style="font-family:Monaco,Courier,monospace;"><span style="font-size:-1pt;"><em># shows the differences in your branch based on the common ancestor for both branches</em></span></span></p> 

<p><span style="font-family:Courier New,courier;"><span style="font-size:11pt;">git diff master...your_branch</span></span></p> 

<p>     </p> 

<p><span style="font-family:Ubuntu;"> If you want to see the different in a branch since you diverted from another branch you can use the ... shortcut. For example if you compare a branch called  <em>your_branch</em> with the  <em>master</em> branch the following command does only show the changes in  <em>your_branch</em> since it diverted from the master branch.</span></p> 

<p> <span style="font-family:Monaco,Courier,monospace;"><span style="font-size:-1pt;"><em># shows the differences between current head of master and your_branch</em></span></span></p> 

<p><span style="font-family:Courier New,courier;"><span style="font-size:11pt;">git diff master your_branch</span></span></p> 

<p>     </p> 

<p><span style="font-family:Ubuntu;">You can see your  <em>remote branches</em> with the following command.</span></p> 

<p> <span style="font-family:Monaco,Courier,monospace;"><span style="font-size:-1pt;"><em># list all remote branches</em></span></span></p> 

<p><span style="font-family:Courier New,courier;"><span style="font-size:11pt;">git branch -r </span></span></p> 

<p>        <span style="font-family:Helvetica,Arial,Droid Sans,sans-serif;"><span style="font-size:-1pt;"> </span></span></p> 

<p><span style="font-family:Ubuntu;"><span style="font-size:11pt;">To see all branches or only the local branches you can use the following commands.</span></span></p> 

<p> <span style="font-family:Monaco,Courier,monospace;"><span style="font-size:-1pt;"><em># list all local branches</em></span></span></p> 

<p><span style="font-family:Courier New,courier;"><span style="font-size:11pt;">git branch</p>

<p>    </p> 

<p><span style="font-family:Monaco,Courier,monospace;"><span style="font-size:-1pt;"><em># delete remote branch from origin</em></span></span></p> 

<p><span style="font-family:Courier New,courier;"><span style="font-size:11pt;">git branch -d -r origin/&lt;remotebranch&gt; </span></span></p>

<p><span style="font-family:Ubuntu;"> If you clone a Git repository, your local  <em>master</em> branch is created as a  <em>tracking branches</em> for  <em>origin/master</em> by Git.</span></p> 

<p>You can create new  <em>tracking branches</em> by specifying the  <em>remote branch</em> during the creation of a branch. The following example demonstrates that.</p> 

<p> <span style="font-family:Monaco,Courier,monospace;"><span style="font-size:-1pt;"><em># setup a tracking branch called newbrach which tracks origin/newbranch</em></span></span></p> 

<p>git checkout -b new branch origin/newbranch</p> 

<p><span style="font-size:11pt;"><span style="font-family:Courier New,courier;">    </span></span><span style="font-family:Helvetica,Arial,Droid Sans,sans-serif;"><span style="font-size:-1pt;"> </span></span></p> 

<p><span style="font-family:Ubuntu;"><span style="font-size:11pt;">To update remote branches which are not setup as tracking branches you use the  </span></span><span style="font-family:Courier New,courier;">git fetch</span><span style="font-family:Ubuntu;"> command which is covered in the next chapter.</span></p> 

<p> The following code shows a few options how you can compare your branches.</p> 

<p> <span style="font-family:Monaco,Courier,monospace;"><span style="font-size:-1pt;"><em># show the long entries between the last local commit and the remote branch  </em></span></span></p>

<p> <span style="font-family:Monaco,Courier,monospace;"><span style="font-size:-1pt;">git pull --rebase origin master </span></span></p>

<p><span style="font-family:Ubuntu;"> 19.2. Fetch vs. pull</span></p> 

<p><p>&nbsp;<p>&nbsp;</p> 

<p>The  <span style="font-family:Courier New,courier;">git pull</span><span style="font-family:Ubuntu;"> command performs a  </span><span style="font-family:Courier New,courier;">git fetch</span><span style="font-family:Ubuntu;"> and  </span><span style="font-family:Courier New,courier;">git merge</span><span style="font-family:Ubuntu;"> (or  </span><span style="font-family:Courier New,courier;">git rebase</span><span style="font-family:Ubuntu;"> based on your Git settings). The </span><span style="font-family:Courier New,courier;">git fetch</span><span style="font-family:Ubuntu;"> does not perform any operations on your local branches. I can always run the fetch command and review the incoming changes.</span></p> 

<p> <strong>20.2. Commands to merge two branches</strong></p> 

<p> <span style="font-family:Monaco,Courier,monospace;"><span style="font-size:-1pt;"><em># syntax: git merge &lt;branch-name&gt; merges into your current selected branch</em></span></span></p> 

<p><span style="font-family:Courier New,courier;"><span style="font-size:11pt;">git merge testing </span></span></p>

<p><span style="font-family:Monaco,Courier,monospace;"><span style="font-size:-1pt;">git tag</span></span></p>

Via the  </span><span style="font-family:Courier New,courier;">-m </span><span style="font-family:Ubuntu;">parameter, you specify the description of this tag. The following command tags the current active HEAD.</span></p> 

<p> <span style="font-family:Monaco,Courier,monospace;"><span style="font-size:-1pt;">git tag version1.6 -m 'version 1.6'</span></span></p> 

<p><span style="font-family:Ubuntu;"><span style="font-size:11pt;">You can also create tags for a certain commit id. </span></span></p> 

<p><span style="font-family:Courier New,courier;">git tag version1.5 -m 'version 1.5'[commit id] </span></p> 

<p>      </p> 

<p><span style="font-family:Ubuntu;"> If you want to use the code associated with the tag, use:</span></p> 

<p><span style="font-family:Monaco,Courier,monospace;"><span style="font-size:-1pt;">git checkout &lt;tag_name&gt; </span></span></p>

<p> <span style="font-family:Monaco,Courier,monospace;"><span style="font-size:-1pt;">git push origin [tagname] </span></span></p>

<p>To see the existing definitions of the remote repositories, use the following command.</p>

<p><span style="font-family:Ubuntu;"><strong>12.2. Push changes to another repository</strong></span></p> 

<p>The  <span style="font-family:Courier New,courier;">git push</span><span style="font-family:Ubuntu;"> command allows you to send data to other repositories. By default it sends data from your current branch to the master branch of the remote repository called  <em>origin</em>.</span></p> 

<p>Make some changes in your local repository and push them from your first repository to the remote repository via the following commands.</p> 

<p> <span style="font-family:Monaco,Courier,monospace;"><span style="font-size:-1pt;"><em># Make some changes in the first repository</em></span></span></p> 

<p><span style="font-family:Courier New,courier;"><span style="font-size:11pt;">cd ~/repo01</span></span></p> 

<p>&nbsp;<p><em># Make some changes in the file</em></p> 

<p>echo &quot;Hello, hello. Turn your radio on&quot; &gt; test01</p> 

<p>echo &quot;Bye, bye. Turn your radio off&quot; &gt; test02</p> 

<p><em># Commit the changes, -a will commit changes for modified files but will not add automatically new files</em></p> 

<p>git commit -a -m &quot;Some changes&quot;</p> 

<p>&nbsp;<p>&nbsp;<em># Push the changes </p>
