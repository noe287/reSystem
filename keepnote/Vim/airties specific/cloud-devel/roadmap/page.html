Cloud çalışması için gerekli  paketler ve akış:
  • air-cloud-agent
    • get_topology (ana method- full topology)
      • sub methods
        • routes(), topology(), friendlyinfo(), steer(), default()   ----->> Bunlar jason formatında output üretecek.
        • LiveView için "default" çağrııyor.  default = routes + topology +  friendly + steer(boş dönecek)
        • friendlyname vs yani friendly için manager içerisinde dhcpcap'in açılması ve bağlı olan her client için bir dosya oluşturması bekleniyor. Cloud agent bunu sorgulayacak daha sonra.
    • Minimim 15-20 parametre aynen lisencingde olduğu gibi GETlenebilir olacak. CDF daemon'ı ve küçük bir CDF dosyasını da kullanıcaz. CDF daemon'ı kullanmamız durumunda uyygulamaya dair değişen bazı config değerlerinden cloud agent otomatik olarak haberdar olacak. Ayrıca bu 15-20 parametrenin GETlenmesi için extra bir işlem yapmayıp lincensing'de olduğu gibi değerleri alabilicez.
    • Getlenmesi gereken bu başlangıç 15-20 değer ve topology datası  için bir JSON dosyası oluşturucaz. Bunu lisencingdeki uzunca dosyanın minimal bir seti olarak düşünebiliriz.
      • Bu json dosaysını hali hazırda licensingin kullandığı bir python scripti bir C dosyasına dönüştürecek. Burada gelen her GET requesti için çağrılması gereken fonksiyon isimleri bulunacak.
        • Gelen requestler uygulama config'ine dair ise CDF file'indan Cihaz configine dair se config.xml'den dönüş olacak. Cihaz datasını sorgulamak için manager üzerinden config.xml sorguları yapabilmemiz gerekiyor. Bunun için de cli librarysini air-cloud-agent'a linlkleyeceğiz.
  • Cloud-Agent Manager Module
    • Starts cloud-agent daemon.
    • Reads cloud-agent config over CDF daemon. Linking libclientcdf.so to be linked here.
      • Bu sayede manager module CDF dosyasında gerekli değeri okuyabilecek. Aslında burada okunması gereken bir tek değer var. Eğer agent enabled değeri TRUE ise manager module'ü agent-daemon'ı başlatacak. O kadar. Geri kalan uygulama config datasını CDF daemon kendisi okuyarak uygulamayı configure edecek ve sistem çalışmaya başlayacak.
  • stats-collector (automesh topology)
    • cloud-agent'ın yaptığı get_topology çağrısını mesh-ng vs. gibi componentlere sorup döndürüyor. Cloud-agent içerisinde de yapılabilir bu döndürme işlemi.
    • automesh için gerekli işlemleri burad yapıp json formatında cloud-agent'a gönderiyor olucaz
  • dhcpcap (friendly info için)
    • manager paketinde bulunuyor. /proc/air_dhcp_clients/ altında her client için dosyalar oluşturuyor. (ip, mac, friendly name) şeklinde.
  • CDF
    • cdf daemon (uygulama configleri için, kolaylaştırıcı ve ileriye dönük integrasyon için)
    • Cihaz configleri config.xml'de olacak eskisi gibi
    • cloud-agent'a dair configler ise küçük bir CDF file'i içerisinde tutucaz. Bu cloud-agent'ın smooth bir şekilde başlamasını kolaylaştıracak.
