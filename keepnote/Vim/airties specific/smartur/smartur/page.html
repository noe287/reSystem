SMARTUR corner cases: 

1) Previously repeated router device had a hidden ssid and we  

changed our repeater's sta_ssid while keeping the hidden-ssid router awake. 

Assume that the new sta_ssid is a dummy one and there is no such device in the air or else it is not ON yet. 

The node will try to scan through bcm_get_target_ap_rssi() function. It will not find that ssid in the air so the first part of the 

if clause will fail. As the while loop iterates over a scannNode-> structure will all type of ssids in it, the second part of the if clause 

that tests whether the ssid is a hidden ssid will be executed. As we have the old sta_mac  in the config.xml it will pass and will measure 

RSSI levels for the older router.
